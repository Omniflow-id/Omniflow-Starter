# 修改密码 ([/admin/change-password](/admin/change-password))

**修改密码**页面允许用户更新其账户密码，并具有全面的策略强制执行以确保安全。

## 快速操作

| 操作 | 方法 |
|------|------|
| 修改密码 | 在 [/admin/change-password](/admin/change-password) 填写表单 |
| 查看个人资料 | 返回 [/admin/profile](/admin/profile) |
| 查看策略要求 | 查看下表 |

## 修改密码步骤

### 如何修改密码

1. 导航到 [/admin/change-password](/admin/change-password)
2. 输入**当前密码**
3. 输入**新密码**（必须满足所有要求）
4. **确认新密码**（必须匹配）
5. 点击**"修改密码"**按钮
6. **成功**：重定向到个人资料页面并显示确认信息

### 表单字段

| 字段 | 必填 | 验证 |
|------|-----|------|
| **当前密码** | ✓ | 必须与您现有的密码匹配 |
| **新密码** | ✓ | 必须满足所有密码策略要求 |
| **确认密码** | ✓ | 必须与新密码完全匹配 |

## 密码策略要求

### 完整策略表

| 要求 | 默认值 | 可配置 | 描述 |
|------|--------|--------|------|
| **最小长度** | 8个字符 | ✓ | 允许的最短密码 |
| **最大长度** | 128个字符 | ✓ | 允许的最长密码 |
| **大写字母** | 必需 (≥ 1) | ✓ | 必须包含 A-Z |
| **小写字母** | 必需 (≥ 1) | ✓ | 必须包含 a-z |
| **数字** | 必需 (≥ 1) | ✓ | 必须包含 0-9 |
| **特殊字符** | 必需 (≥ 1) | ✓ | 必须包含 !@#$%^&* 等 |
| **最少数字数** | 1 | ✓ | 最少数字位数 |
| **最少符号数** | 1 | ✓ | 最少特殊字符位数 |
| **最大重复字符** | 3 | ✓ | 最多连续相同字符数 |
| **禁用模式** | 可配置 | ✓ | 禁止的词语/模式 |

### 环境变量

管理员可以通过 `.env` 自定义策略：

```env
PASSWORD_MIN_LENGTH=8
PASSWORD_MAX_LENGTH=128
PASSWORD_REQUIRE_UPPERCASE=true
PASSWORD_REQUIRE_LOWERCASE=true
PASSWORD_REQUIRE_NUMBERS=true
PASSWORD_REQUIRE_SYMBOLS=true
PASSWORD_MIN_SYMBOLS=1
PASSWORD_MIN_NUMBERS=1
PASSWORD_MAX_REPEATING=3
PASSWORD_FORBIDDEN_PATTERNS=password,admin,12345,qwerty
```

## 密码验证规则

### 字符类型要求

**大写字母 (A-Z):**
- 至少需要1个大写字母
- 示例：`A`、`Z`、`M`

**小写字母 (a-z):**
- 至少需要1个小写字母
- 示例：`a`、`z`、`m`

**数字 (0-9):**
- 至少需要1个数字
- 示例：`0`、`5`、`9`

**特殊字符：**
- 至少需要1个特殊字符
- 允许的字符：`` !@#$%^&*()_+-=[]{}|;':",./<>?`~ ``
- 示例：`!`、`@`、`#`、`$`

### 高级规则

**最大连续重复字符：**
- 最多允许3个连续相同字符
- ✅ 好：`AAA`、`111`、`!!!`
- ❌ 差：`AAAA`、`1111`、`!!!!`

**禁用模式：**
- 不得包含常用词语（可配置）
- 默认禁用词：`password`、`admin`、`12345`、`qwerty`
- 匹配时不区分大小写

### 密码强度示例

**✅ 强密码：**
- `MyP@ssw0rd2024`（混合大小写、数字、符号）
- `Secure!123Pass`（满足所有要求）
- `Tr0ng#P@ssword`（复杂度好）

**❌ 弱密码：**
- `password`（禁用模式）
- `12345678`（没有字母，太简单）
- `Password`（缺少数字、符号）
- `AAAAA123!`（太多重复字符）
- `pass`（太短）

## 验证错误信息

### 常见错误

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| Password must be at least 8 characters | 太短 | 使用至少8个字符 |
| Password must contain uppercase | 缺少 A-Z | 添加至少一个大写字母 |
| Password must contain lowercase | 缺少 a-z | 添加至少一个小写字母 |
| Password must contain numbers | 缺少 0-9 | 添加至少一个数字 |
| Password must contain symbols | 缺少特殊字符 | 添加 !@#$%^&* 等 |
| Too many repeating characters | AAAA 模式 | 将连续重复限制为3个 |
| Password contains forbidden pattern | 禁用词 | 避免使用"password"等常用词 |
| Passwords do not match | 确认不匹配 | 确认密码必须与新密码匹配 |
| Current password incorrect | 当前密码错误 | 输入正确的当前密码 |

## 生成的密码（管理员批量导入）

### 批量上传的密码模式

当管理员通过Excel导入用户时，密码会自动生成：

**模式：** `FullNameWithoutSpaces@12345?.`

**示例：**
- Eric Julianto → `EricJulianto@12345?.`
- Jane Smith → `JaneSmith@12345?.`
- Ahmad Wijaya → `AhmadWijaya@12345?.`

**策略合规性：**
- ✅ 大写字母：每个名字部分的首字母
- ✅ 小写字母：其余字母
- ✅ 数字：`12345`
- ✅ 符号：`@`、`?`、`.`
- ✅ 满足最小长度要求（8个字符以上）

### 生成密码的安全警告

⚠️ **新用户须知：**
- 生成的密码**可预测**
- 用户应在**首次登录后立即修改**
- 管理员可以在 [/admin/passwords](/admin/passwords) 查看生成的密码
- **请勿**不安全地分享生成的密码

## 常见场景

### 场景1：忘记当前密码

**问题：** 不记得当前密码，无法修改

**解决方案：**
1. 注销您的账户
2. 联系系统管理员
3. 管理员可以在 [/admin/users](/admin/users) 重置您的密码
4. 管理员提供临时密码
5. 使用临时密码登录
6. 立即更改为您自己的密码

**注意：** 系统不支持自助密码重置功能（安全特性）。

### 场景2：密码被策略拒绝

**问题：** 新密码一直被拒绝

**诊断：**
1. 检查上表中的所有策略要求
2. 仔细查看错误信息
3. 使用密码强度检查器（如果可用）

**解决方案：**
- 确保至少8个字符
- 包含大写字母 (A-Z)
- 包含小写字母 (a-z)
- 包含数字 (0-9)
- 包含特殊字符 (!@#$)
- 避免重复字符（最多3个）
- 避免禁用词

### 场景3：批量导入后首次登录

**对于由管理员创建的新用户：**

1. 查看邮箱/文档获取临时密码
2. 使用生成的密码登录（格式：`FullName@12345?.`）
3. **立即导航到** [/admin/change-password](/admin/change-password)
4. 更改为安全的个人密码
5. 安全保存密码（建议使用密码管理器）

### 场景4：定期更新密码

**最佳实践计划：**

1. **每90天**更改一次密码（推荐）
2. 导航到 [/admin/change-password](/admin/change-password)
3. 输入当前密码
4. 选择一个强密码（与之前的不同）
5. 更新密码管理器条目
6. 注销并重新登录以验证

## 故障排除

| 问题 | 诊断 | 解决方案 |
|------|------|----------|
| 表单验证失败 | 检查所有策略要求 | 查看错误信息，确保满足所有规则 |
| 当前密码错误 | 拼写错误或大小写锁定 | 验证正确的密码，检查大小写锁定 |
| 确认不匹配 | 输入错误 | 小心重新输入确认密码 |
| 密码太弱 | 密码简单 | 增加复杂度：混合大小写、数字、符号 |
| 无法访问页面 | 未登录 | 先登录，然后访问修改密码 |
| 更改未保存 | 服务器/网络错误 | 检查网络连接，重试 |

## 安全最佳实践

### 选择强密码

✅ **好做法：**
- 使用密码管理器生成随机密码
- 每个系统使用唯一密码
- 使用密码短语（示例：`Coffee$Morning2024!`）
- 在整个密码中混合字符类型
- 创建易记但不可预测的密码

❌ **避免：**
- 个人信息（生日、姓名、电话）
- 常见模式（123456、qwerty、password）
- 字典词语
- 在不同系统中重复使用密码
- 与任何人分享密码

### 密码维护

✅ **定期更新：**
- 每90天更改一次密码
- 如果出现以下情况立即更新：
  - 怀疑密码泄露
  - 分享凭据后（临时访问）
  - 离职时
  - 宣布安全漏洞时
  - 从承包商转为员工时

### 活动日志

所有密码更改都会记录到活动系统中：
- 更改时间戳
- 修改密码的用户
- IP地址
- 成功/失败状态

**查看日志：** [/admin/log](/admin/log)（仅管理员）
