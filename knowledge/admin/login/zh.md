# 登录流程

**登录**页面是进入 Omniflow 系统的主要入口。该页面处理用户身份验证，使用密码和双因素身份验证 (2FA) 提供分层安全性。

## 使用者

- **所有系统用户** - 在 Omniflow 中拥有账户的员工、经理和管理员
- **新用户** - 由管理员新注册的员工
- **承包商/外部人员** - 获得临时系统访问权限的外部方

## 使用时机

1. **首次访问** - 账户创建后首次进入系统
2. **会话结束** - 注销后会话超时（24小时无活动）
3. **从新设备登录** - 从新的浏览器/设备访问
4. **密码重置** - 管理员重置密码后，用户需要重新登录

## 登录步骤

### 1. 输入凭据

在登录页面，输入：
- **邮箱**: 系统注册的邮箱地址
- **密码**: 您的账户密码

### 2. 验证密码

系统将验证：
- 邮箱已在系统中注册
- 密码与存储的密码匹配
- 账户处于活跃状态

### 3. 双因素身份验证 (2FA)

如果密码正确，系统将向您的邮箱发送 OTP 验证码：
- 密码验证后自动发送 OTP
- 您将被重定向到 OTP 验证页面
- 输入发送到邮箱的 6 位数字验证码

### 4. 访问系统

OTP 验证后，您将被重定向到主页（仪表板）。

## 错误信息和解决方案

| 错误信息 | 原因 | 解决方案 |
|-------------|----------|-----------|
| 邮箱或密码错误 | 邮箱未注册或密码错误 | 检查邮箱和密码，如果忘记请联系管理员 |
| 账户未激活 | 账户被管理员停用 | 联系管理员重新激活 |
| 登录尝试过多 | 达到速率限制 | 等待15分钟后重试 |
| 会话结束 | 空闲时间过长 | 重新登录 |

## 登录安全

### 应用的安全保护

1. **密码加密** - 密码使用强加密存储 (bcrypt)
2. **双因素身份验证** - OTP 验证码发送到邮箱进行额外验证
3. **速率限制** - 通过限制登录尝试防止暴力攻击
4. **活动日志** - 所有登录尝试（成功/失败）都会被记录以供审计

### 安全最佳实践

✅ **应该做：**
- 使用在其他系统上未使用的唯一密码
- 永不共享登录凭据
- 离开电脑时注销
- 立即报告可疑活动

❌ **应该避免：**
- 在浏览器中保存密码（尤其是在共享电脑上）
- 在没有 VPN 的公共 WiFi 上登录
- 在共享电脑上使用"记住我"功能
- 忽略安全警告

## 会话设置

### 会话持续时间

- **生命周期**: 24 小时（如果无活动）
- **自动续订**: 有活动时自动延展会话
- **超时警告**: 会话结束前 2 分钟出现

### 会话安全机制

- 每次活动（点击、输入）都会更新会话计时器
- 会话绑定到浏览器和 IP 地址
- 如果 IP 发生重大变化，会话将结束

## 故障排除

| 问题 | 解决方案 |
|---------|--------|
| 未收到 OTP 邮箱 | 检查垃圾邮件文件夹，确保邮箱正确，如果仍有问题请联系管理员 |
| 邮箱未注册 | 联系管理员注册账户 |
| 忘记密码 | 联系管理员重置密码 |
| 账户被锁定 | 联系管理员解锁 |
| 完全无法登录 | 与管理员确认账户状态 |

## 与其他功能的集成

### 登录成功后

- **权限已加载** - 系统根据您的角色加载访问权限
- **重定向到仪表板** - 您根据角色进入主页
- **活动已记录** - 创建登录活动以进行审计跟踪

### 如果登录失败

- 失败的尝试会记录 IP 和时间详情
- 管理员可以在活动日志中查看登录历史
- 如果失败次数过多，账户可能需要管理员验证
