# 活动日志 ([/admin/log/index](/admin/log/index))

**活动日志**页面提供完整的审计跟踪，记录Omniflow系统中所有发生的活动。

## 主要功能

### 1. 活动日志列表
完整的表格，包含以下列：
- **时间戳**: 事件发生时间
- **用户**: 执行操作的人员
- **操作类型**: 活动类型（创建、更新、删除、登录等）
- **模块**: 受影响的系统区域
- **详情**: JSON格式的附加详细信息
- **IP地址**: 用户IP地址（用于安全审计）

### 2. 筛选和搜索

**筛选条件：**
- **日期范围**: 起始日期 - 结束日期
- **用户**: 选择特定用户
- **操作类型**: 创建、更新、删除、登录、登出、错误
- **模块**: 用户、权限、队列、缓存等

**搜索：**
- 按活动描述搜索
- 在JSON详情中搜索

### 3. 导出日志

**导出格式：**
- **CSV**: 用于Excel分析
- **JSON**: 用于其他系统集成
- **PDF**: 用于审计报告

**筛选后导出：**
- 仅导出当前筛选结果
- 适用于合规报告

## 操作类型

| 操作 | 说明 | 示例 |
|------|------|------|
| **CREATE** | 创建新数据 | 添加用户、创建角色 |
| **UPDATE** | 修改数据 | 编辑用户、更新权限 |
| **DELETE** | 删除数据 | 删除用户、软删除 |
| **LOGIN** | 用户登录系统 | 管理员从IP x.x.x.x登录 |
| **LOGOUT** | 用户登出 | 会话结束 |
| **ERROR** | 错误/异常 | 登录失败尝试 |
| **VIEW** | 读取数据访问 | 查看用户详情 |

## 追踪模块

- **users**: 用户管理活动
- **permissions**: 权限和角色变更
- **queue**: 任务队列操作
- **cache**: 缓存刷新/清除
- **ai**: AI功能使用（聊天、助手、Copilot）
- **system**: 系统配置

## 元数据详情

点击日志行查看完整的JSON详情：
```json
{
  "user_id": 7,
  "username": "admin",
  "action": "UPDATE",
  "module": "users",
  "record_id": 15,
  "changes": {
    "old": { "role": "User" },
    "new": { "role": "Manager" }
  },
  "ip_address": "192.168.1.100"
}
```

## 审计用例

### 安全调查
**场景：** 发现可疑活动

**步骤：**
1. 筛选可疑用户
2. 筛选事件日期范围
3. 检查来自未知IP的登录操作
4. 检查权限/角色变更
5. 导出日志作为证据

### 合规报告
**场景：** 需要月度审计报告

**步骤：**
1. 设置上月的日期范围
2. 筛选关键操作（创建、删除）
3. 筛选敏感模块（权限、用户）
4. 导出为PDF
5. 提交给审计员

### 故障排除
**场景：** 数据被修改但不知道是谁做的

**步骤：**
1. 按record_id搜索
2. 筛选相关模块
3. 检查更新/删除操作
4. 查看执行操作的用户详情
5. 检查时间戳了解时间线

## 数据保留和清理

**默认保留：**
- 日志保留90天
- 每周自动运行清理任务
- 归档到冷存储（如果已配置）

**手动清理：**
- 拥有`manage_logs`权限的管理员可以手动清理
- 筛选旧日期范围 → 删除
- 或在删除前先导出

## 权限

| 权限 | 访问权限 |
|------|----------|
| `view_logs` | 查看日志列表 |
| `export_logs` | 导出日志数据 |
| `manage_logs` | 清理/归档日志 |

## 最佳实践

### 审计跟踪
✅ **始终启用日志记录：**
- 权限变更
- 用户增删改
- 系统配置变更
- 敏感数据访问

### 定期审查
✅ **制定日志审查计划：**
- 每日：检查错误/异常
- 每周：审查失败登录
- 每月：审计权限变更

### 安全注意事项
⚠️ **重要提示：**
- 普通用户无法删除日志
- 超级管理员可以查看所有日志
- 导出的日志包含敏感数据 → 请谨慎处理

## 故障排除

| 问题 | 解决方案 |
|------|----------|
| 日志不显示 | 检查当前筛选条件，清除筛选 |
| 导出失败 | 检查数据大小，尝试更具体的筛选 |
| JSON详情错误 | 检查浏览器控制台，刷新页面 |
| 性能慢 | 使用更小的日期范围 |
