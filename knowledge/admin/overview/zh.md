# 用户概览 ([/admin/overview](/admin/overview))

**用户概览**页面提供 Omniflow 系统中用户账户的统计信息和分析洞察。

## 快速操作

| 操作 | 方式 |
|------|------|
| 查看用户统计 | 访问 [/admin/overview](/admin/overview) |
| 查看所有用户 | 访问 [/admin/users](/admin/users) |
| 管理角色 | 访问 [/admin/roles](/admin/roles) |
| 查看活动日志 | 访问 [/admin/log](/admin/log) |

## 注意路由冲突

⚠️ **重要：** 此页面服务**两个路由**：
- `/admin/overview` - 模块概览
- `/admin/overview` - 用户特定统计

两个路由显示相同的用户统计内容。这是系统已知的路由冲突。

## 用户统计仪表盘

### 用户总数指标

| 指标 | 描述 |
|------|------|
| **用户总数** | 系统中的所有用户账户 |
| **活跃用户** | `is_active = true` 的用户 |
| **非活跃用户** | `is_active = false` 的用户 |
| **本月新增** | 本月创建的用户 |

### 角色分布

**各角色用户数：**
- **管理员**：拥有完全系统访问权限的用户
- **经理**：拥有中级访问权限的用户
- **用户**：拥有基本访问权限的用户
- **自定义角色**：部门特定角色

### 账户状态细分

**活跃 vs 非活跃：**
- 活跃账户的百分比
- 非活跃账户的数量
- 最新的激活/停用趋势

## 统计可视化

### 推荐的图表

**用户增长图表（折线图）：**
- X轴：月份
- Y轴：新用户数
- 显示每月用户注册趋势

**角色分布（饼图）：**
- 分段：每个角色
- 百分比：各角色分布比例
- 颜色：管理员（红色）、经理（黄色）、用户（蓝色）

**活跃状态（环形图）：**
- 内圈：活跃数量
- 外圈：非活跃数量
- 百分比：活跃与非活跃比例

页面使用 Chart.js 进行数据可视化：

## 常见场景

### 场景一：月度用户审计

**审核步骤：**
1. 访问 [/admin/overview](/admin/overview)
2. 检查**用户总数**
3. 审核**活跃与非活跃**比例
4. 记录**本月新增**数量
5. 与上月进行比较
6. 记录以供合规使用

### 场景二：角色分布分析

**分析方法：**
1. 查看角色分布图表
2. 识别不平衡的角色：
   - 管理员过多 → 安全风险
   - 经理过少 → 瓶颈
   - 自定义角色数量异常
3. 导航到 [/admin/roles](/admin/roles) 查看详情
4. 在 [/admin/users](/admin/users) 审核角色分配

### 场景三：清理非活跃账户

**清理流程：**
1. 记录**非活跃用户**数量
2. 导航到 [/admin/users](/admin/users)
3. 筛选或识别非活跃账户
4. 审核最后登录日期（如有）
5. 停用或删除不使用的账户
6. 返回概览页面验证减少数量

### 场景四：监控增长

**月度审核：**
1. 检查**本月新增**指标
2. 与目标比较（例如：每月50个新用户）
3. 如果低于目标：
   - 审核入职流程
   - 检查批量导入是否成功
   - 验证注册功能可用性
4. 如果高于目标：
   - 验证注册是否合法
   - 检查是否有批量导入
   - 审核系统容量

## 统计上下文

### 缓存集成

用户统计会被缓存：
- **缓存键**：`admin:user:overview` 或 `admin:overview`
- **有效期**：5分钟
- **失效**：用户 CRUD 操作后

**查看缓存性能：** [/admin/cache/stats](/admin/cache/stats)

### 数据源

**数据库查询：**
```sql
-- 用户总数
SELECT COUNT(*) FROM users WHERE deleted_at IS NULL

-- 活跃用户
SELECT COUNT(*) FROM users WHERE is_active = true AND deleted_at IS NULL

-- 本月新增
SELECT COUNT(*) FROM users
WHERE created_at >= FIRST_DAY_OF_MONTH
AND deleted_at IS NULL

-- 角色分布
SELECT role, COUNT(*) FROM users
WHERE deleted_at IS NULL
GROUP BY role
```

### 活动日志

用户操作会被记录：
- 用户创建
- 账户激活/停用
- 角色变更
- 用户删除

**查看日志：** [/admin/log](/admin/log)

## 故障排除

| 问题 | 检查 | 解决方案 |
|------|------|---------|
| 统计不加载 | 缓存状态 | 在 [/admin/cache/stats](/admin/cache/stats) 清除缓存 |
| 数量显示错误 | 软删除 | 验证 `deleted_at IS NULL` 过滤器 |
| 新用户数量错误 | 时区 | 检查服务器时区配置 |
| 角色分布缺失 | 数据库查询 | 验证角色表已填充 |
| 图表不渲染 | 浏览器控制台 | 检查 Chart.js 库是否加载 |

## 性能考虑

### 查询优化

统计查询：
- **已索引**：状态字段的 COUNT 查询快速
- **已缓存**：5分钟 TTL 用于重复访问
- **轻量级**：仅聚合计数，无用户数据

### 缓存优势

- **响应时间**：< 5ms（缓存）vs 50-100ms（数据库）
- **数据库负载**：高流量时减少95%
- **可扩展性**：处理数千个并发视图

## 最佳实践

### 常规监控

✅ **每周：**
- 审核用户总数
- 检查活跃/非活跃比例
- 监控新用户趋势
- 验证角色分布

✅ **每月：**
- 审计非活跃账户
- 审核增长指标
- 与业务目标比较
- 记录以供报告使用

### 告警阈值

设置监控：
- **非活跃比例 > 30%** - 非活跃账户过多
- **新用户 = 0** - 注册问题
- **管理员数量 > 10** - 需要安全审核
- **用户总数 > 容量** - 扩展基础设施

### 与报告集成

使用概览统计：
- **管理报告**：月度 KPI
- **合规审计**：用户账户跟踪
- **容量规划**：增长预测
- **安全审核**：角色分布分析
