# 用户资料 ([/admin/profile](/admin/profile))

**用户资料**页面展示您的账户信息、安全设置，并提供密码管理入口。

## 快速操作

| 操作 | 方法 |
|------|------|
| 查看资料 | 访问 [/admin/profile](/admin/profile) |
| 修改密码 | 点击 **"Change Password"** → 跳转至 [/admin/change-password](/admin/change-password) |
| 查看2FA状态 | 查看 **"2FA Security"** 区域 |
| 检查账户状态 | 查看 **"Account Status"** 标识 |

## 资料信息

### 账户详情

| 字段 | 说明 |
|------|------|
| **Username** | 您的登录用户名（唯一） |
| **Email** | 账户邮箱地址 |
| **Full Name** | 您的全名 |
| **Role** | 当前角色（Admin、Manager、User） |
| **Created At** | 账户创建时间戳 |

### 账户状态

- **🟢 Active**：完全系统访问权限
- **🔴 Inactive**：账户已停用，无法访问

**注意：** 只有管理员可以在 [/admin/users](/admin/users) 中修改账户状态。

## 两步验证（2FA）

### 当前2FA方式

系统使用**邮箱OTP**进行两步验证：

- **方式**：一次性密码发送至您的邮箱
- **有效期**：每个OTP有效期5分钟
- **非阻塞**：邮件通过RabbitMQ队列发送（响应时间<200ms）
- **备选**：队列不可用时使用同步邮件发送

### 2FA安全功能

✅ **安全优势：**
- 除密码外的额外保护层
- 防止未授权访问
- 每次登录验证邮箱
- 基于会话的OTP存储

### 开发环境跳过

开发/测试环境可设置：
- **`DEV_2FA_BYPASS=true`** 环境变量
- 跳过OTP验证
- **请勿在生产环境使用**

## 安全设置

### 密码策略

您的密码必须满足以下要求：

| 要求 | 默认值 |
|------|--------|
| 最小长度 | 8个字符 |
| 最大长度 | 128个字符 |
| 大写字母 | 必需 |
| 小写字母 | 必需 |
| 数字 | 必需（至少1个） |
| 特殊字符 | 必需（至少1个） |
| 最大连续重复字符 | 3个连续 |

**修改密码：** [/admin/change-password](/admin/change-password)

### 会话安全

- **会话超时**：24小时无活动后超时
- **滑动会话**：活动时自动续期
- **不活跃提醒**：超时前2分钟弹出提示框
- **保持活跃**：用户活动时后台刷新会话

### 活动日志

您的所有账户活动都会被记录：
- 登录尝试
- 密码变更
- 权限变更
- 资料更新

**查看日志：** [/admin/log](/admin/log)（按您的用户名筛选）

## 常见场景

### 场景1：修改密码

**何时修改：**
- 定期更新安全信息（建议每90天）
- 怀疑账户可能被入侵
- 临时分享凭证后
- 入职/离职

**修改方法：**
1. 点击资料页的 **"Change Password"** 按钮
2. 跳转至 [/admin/change-password](/admin/change-password)
3. 输入当前密码
4. 输入新密码（需符合策略）
5. 确认新密码
6. 提交 → 密码已更新

### 场景2：查看您的权限

**查看方法：**
1. 您的角色决定您的权限
2. 在 [/admin/roles](/admin/roles) 查看角色详情
3. 管理员可以在 [/admin/users](/admin/users) 授予/撤销个人权限

**各角色常见权限：**
- **Admin**：所有权限
- **Manager**：`view_users`、`manage_users`、`view_logs`、`view_profile`
- **User**：仅 `view_profile`

### 场景3：2FA收不到OTP

**故障排除：**
1. 检查邮件的垃圾邮件/广告邮件文件夹
2. 确认资料中的邮箱地址正确
3. 如是管理员，检查 [/admin/queue](/admin/queue) 队列状态
4. 如持续出现问题，联系系统管理员

**管理员故障排除：**
- 检查RabbitMQ连接状态
- 查看邮件worker日志
- 验证SMTP配置
- 检查熔断器状态

### 场景4：账户被锁定/停用

**症状：**
- 无法登录
- 提示"Account inactive"错误
- 资料页显示 🔴 Inactive 状态

**解决方案：**
- 联系系统管理员
- 管理员可在 [/admin/users](/admin/users) 重新激活
- **注意：** 您无法自行激活账户

## 故障排除

| 问题 | 检查 | 解决方案 |
|------|------|----------|
| 无法修改密码 | 当前密码是否正确？ | 验证当前密码，确认符合密码策略 |
| 2FA OTP未收到 | 邮箱地址是否正确？ | 检查垃圾邮件夹，联系管理员 |
| 账户显示inactive | 账户状态标识 | 联系管理员重新激活 |
| 权限不足错误 | 您的角色权限 | 联系管理员审核权限 |
| 会话超时太快 | 活动频率 | 系统会在活动时保持会话 |

## 安全最佳实践

### 密码管理

✅ **推荐做法：**
- 为本系统使用独立密码
- 每90天更换一次
- 不分享凭证
- 使用密码管理器

❌ **避免：**
- 重复使用其他系统的密码
- 简单/可预测的密码
- 共享账户访问
- 记录密码

### 会话安全

✅ **推荐做法：**
- 离开电脑时登出
- 共享电脑上不保存密码
- 报告可疑活动
- 保护邮箱安全（用于2FA）

❌ **避免：**
- 在共享电脑上保持会话开启
- 无VPN使用公共WiFi
- 分享2FA验证码
- 忽略安全警告

### 账户安全

✅ **监控：**
- 定期查看您的活动日志
- 检查不熟悉的登录位置
- 确认邮箱地址是最新的
- 立即报告可疑活动
