# 角色管理 ([/admin/roles](/admin/roles))

**角色管理**功能允许管理员使用基于角色的访问控制（RBAC）管理 Omniflow 系统中用户的访问级别。

## 快速操作

| 操作 | 方法 |
|------|------|
| 查看所有角色 | 访问 [/admin/roles](/admin/roles) |
| 创建新角色 | 点击 **"添加角色"** 按钮 |
| 编辑权限 | 点击角色 → **管理权限** |
| 删除角色 | 点击角色 → 删除（如果没有用户） |

## 默认角色

| 角色 | 描述 | 无法删除 |
|------|------|----------|
| **Admin** | 完整系统访问权限 | ✓ 已锁定 |
| **Manager** | 用户管理和监控 | ✗ |
| **User** | 仅基本 profile 访问 | ✗ |

## 创建新角色

1. 访问 **[角色管理](/admin/roles)**
2. 点击 **"+ 添加角色"** 按钮
3. 填写表单：
   - **角色名称**：唯一名称（例如："人力资源经理"、"财务"）
   - **描述**：该角色的简要用途
4. 点击 **保存** → 角色创建完成，权限为空
5. **重要**：创建后请立即分配权限

## 管理权限

创建角色后，您需要分配权限：

1. 在 [/admin/roles](/admin/roles) 点击角色
2. 点击 **"管理权限"** 按钮
3. 从列表中选择权限：
   - `view_users` - 查看用户列表
   - `manage_users` - 用户 CRUD + 重置密码
   - `manage_permissions` - 配置角色
   - `view_logs` - 读取活动日志
   - `manage_cache` - 刷新 Redis 缓存
   - `manage_queue` - 管理任务队列
   - `view_profile` - 查看自己的 profile
4. 点击 **保存权限**

## 权限参考

| 权限 | 描述 | 用途 |
|------|------|------|
| `view_users` | 查看用户账户 | 经理需要 |
| `manage_users` | 用户 CRUD | HR/管理员任务 |
| `manage_permissions` | 配置角色 | 仅限超级管理员 |
| `view_logs` | 读取活动历史 | 审计和监控 |
| `manage_cache` | 刷新 Redis 缓存 | 技术操作 |
| `manage_queue` | 管理任务队列 | 后台任务 |
| `view_profile` | 查看自己的 profile | 所有用户都需要 |

## 编辑角色

### 修改角色名称/描述
1. 在 [/admin/roles](/admin/roles) 点击角色
2. 点击 **编辑**
3. 更新字段
4. **注意**：权限分配保持不变

### 修改权限
1. 点击角色
2. 点击 **管理权限**
3. 添加/删除权限
4. **注意**：更改立即对所有使用该角色的用户生效

## 删除角色 ⚠️

**删除前：**
1. 检查是否有用户被分配该角色在 [/admin/users](/admin/users)
2. 先将用户重新分配到其他角色
3. 仅在没有用户使用该角色时删除

**删除方法：**
1. 点击角色
2. 点击 **删除**
3. 确认删除

**安全规则：**
- ❌ **Admin** 角色（ID 1）无法删除
- ❌ 如果有用户被分配该角色，则无法删除
- ✅ 删除的角色的权限会从所有用户中移除

## 常见场景

### 场景 1：新部门需要访问权限
**问题：** 财务团队需要访问用户数据
**解决方案：**
1. 在 [/admin/roles](/admin/roles) 创建 "Finance" 角色
2. 分配权限 `view_users`、`view_profile`
3. 在 [/admin/users](/admin/users) 将财务用户添加到此角色

### 场景 2：经理需要额外权限
**问题：** 经理需要查看日志
**解决方案：**
1. 进入 [/admin/roles](/admin/roles)
2. 点击 Manager 角色 → 管理权限
3. 添加权限 `view_logs`
4. 保存

### 场景 3：快速撤销访问权限
**问题：** 员工今天离职
**解决方案：**
1. 进入 [/admin/users](/admin/users)
2. 搜索用户 → 编辑
3. 将 "激活" 设置为关闭
4. 用户立即失去访问权限

## 故障排除

| 问题 | 解决方案 |
|------|----------|
| 用户无法访问功能 | 检查角色权限在 [/admin/roles](/admin/roles) |
| 权限过多 | 从角色中移除不需要的权限 |
| Admin 被锁定 | 联系系统管理员 |
| 权限不生效 | 用户可能需要重新登录 |
