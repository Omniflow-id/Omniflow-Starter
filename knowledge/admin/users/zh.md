# 用户管理 (/admin/users)

**用户管理**页面是管理 Omniflow 系统用户账户的核心控制中心。

## 主要功能

### 1. 用户列表
完整表格包含以下信息：
- **ID**：用户唯一标识
- **用户名**：用户登录名
- **邮箱**：注册邮箱地址
- **全名**：用户完整姓名
- **角色**：访问权限级别（管理员、经理、用户）
- **状态**：激活/停用
- **最后登录**：上次登录时间
- **创建时间**：账户创建日期

### 2. 添加新用户

**手动添加：**
1. 点击 **"添加用户"** 按钮
2. 填写表单：
   - 用户名（唯一）
   - 邮箱（有效且唯一）
   - 全名
   - 密码（或自动生成）
   - 角色
3. 点击 **保存**

**自动生成密码：**
- 系统将生成随机密码
- 创建时仅显示一次密码
- 用户首次登录时必须更改密码

### 3. 编辑用户

**可编辑的数据：**
- 邮箱
- 全名
- 角色
- 状态（激活/停用）
- 密码（重置）

**编辑方法：**
1. 点击用户行的 **编辑** 图标
2. 修改所需数据
3. 点击 **保存**

### 4. 重置密码

**重置方法：**
1. 点击 **编辑** 用户
2. 点击 **"重置密码"** 按钮
3. 选择：
   - **自动生成**：系统创建新密码
   - **手动**：管理员输入新密码
4. 显示新密码（复制并发送给用户）

### 5. 切换激活状态

**停用用户：**
1. 点击状态列的开关按钮
2. 确认停用
3. 用户将无法登录

**重新激活：**
1. 点击开关按钮
2. 用户可以使用相同凭证重新登录

### 6. 删除用户（软删除）

**删除方法：**
1. 点击 **删除** 图标（垃圾桶）
2. 确认删除
3. 用户被软删除（可恢复）

**注意：** 用户数据不会真正丢失，仅标记为已删除。

### 7. 批量导入Excel

**下载模板：**
1. 点击 **"下载模板"**
2. 用用户数据填写模板
3. 上传Excel文件

**模板列：**
- username（必填，唯一）
- email（必填，有效）
- full_name（必填）
- password（选填，为空则自动生成）
- role（必填：Admin/Manager/User）
- is_active（1 或 0）

**导入验证：**
- 检查用户名/邮箱重复
- 验证邮箱格式
- 手动输入密码最少8个字符
- 角色必须有效

### 8. 导出数据

**导出选项：**
- **CSV**：导入其他系统
- **Excel**：可编辑格式
- **JSON**：用于API集成

**筛选导出：**
- 仅导出筛选后的数据
- 示例：仅导出活跃用户

## 筛选与搜索

### 搜索框
按以下条件搜索：
- 用户名
- 邮箱
- 全名

### 列筛选
- **角色**：按角色筛选
- **状态**：激活/停用/全部
- **日期范围**：账户创建日期

### 排序
点击列标题进行排序：
- ID（升序/降序）
- 用户名（A-Z/Z-A）
- 创建时间（最新/最早）
- 最后登录（活跃/不活跃）

## 所需权限

| 权限 | 描述 |
|------------|-----------|
| `view_users` | 查看用户列表 |
| `manage_users` | 用户CRUD操作 |
| `manage_permissions` | 修改用户权限 |

## 最佳实践

### 安全性
✅ **建议：**
- 为新用户使用自动生成密码
- 要求用户首次登录时更改密码
- 离职员工应停用而非删除
- 记录权限变更审计日志

❌ **禁止：**
- 将他人密码告诉任何人
- 用户没有角色
- 删除具有重要数据的用户

### 管理
✅ **推荐：**
- 每月审查活跃用户
- 停用90天未登录的用户
- 根据工作职能分配角色
- 记录权限变更文档

## 问题排查

| 问题 | 解决方案 |
|---------|--------|
| 用户无法登录 | 检查激活状态和密码 |
| 邮箱已被注册 | 使用其他邮箱或重置旧用户 |
| 角色不显示 | 检查角色页面配置 |
| 导入失败 | 检查Excel格式和验证错误信息 |
| 密码重置不工作 | 确保邮件服务器正常运行 |
