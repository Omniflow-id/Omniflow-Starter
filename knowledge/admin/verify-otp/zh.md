# 验证码验证

**OTP验证码验证**页面是Omniflow系统登录过程中的第二步安全验证。在成功验证密码后，系统会要求您输入发送到邮箱的验证码。

## 适用人群

- **所有登录用户** - 成功验证密码后都需要经过此步骤
- **活跃账户用户** - 未被禁用的账户
- **首次登录用户** - 重置密码后仍需进行2FA验证

## 使用场景

1. **正常登录** - 密码验证成功后
2. **密码重置后登录** - 使用新密码登录
3. **新设备登录** - 系统检测到新设备
4. **会话过期** - 会话超时后重新登录需要OTP验证码

## 使用方法

### 1. 查看邮件

跳转到OTP验证码页面后：
- 打开您的邮箱收件箱
- 查找来自Omniflow系统的邮件
- OTP验证码由6位数字组成

### 2. 输入验证码

- 在提供的输入框中输入6位数字验证码
- 确保数字之间没有空格
- 点击"验证"按钮或按回车键

### 3. 等待处理

- 系统会在几秒内验证验证码
- 验证成功后将跳转到仪表板
- 验证失败会显示错误信息

## 验证码有效期

### 有效期时长

- **时长**：验证码发送后5分钟内有效
- **计时器显示**：系统显示剩余可用时间
- **过期处理**：过期后需从头重新登录

### OTP过期后怎么办

1. 系统会返回登录页面
2. 需要重新输入邮箱和密码
3. 系统会重新发送新的验证码到邮箱

## 错误信息及解决方案

| 错误信息 | 原因 | 解决方案 |
|-------------|----------|--------|
| 验证码无效 | 输入错误或已过期 | 查看邮箱中最新的验证码，不要使用旧验证码 |
| OTP已过期 | 超过5分钟 | 重新登录获取新验证码 |
| 尝试次数过多 | 多次输入错误验证码 | 等待5分钟后重试 |
| 会话未找到 | 刷新页面或导航无效 | 从登录页面重新登录 |

## 安全机制

### 为什么需要OTP？

- **额外的安全层** - 即使密码泄露，账户仍然安全
- **验证邮箱所有权** - 确保是账户合法所有者访问
- **防止未授权访问** - 攻击者必须同时拥有邮箱访问权限

### OTP安全特性

- 验证码是唯一的随机生成
- 仅适用于单个登录会话
- 无法重复使用
- 每次登录尝试的验证码都不同

## 安全建议

### ✅ 正确做法

- 切勿向任何人分享OTP验证码
- 系统永远不会通过电话要求提供OTP验证码
- 如果收到非本人请求的OTP邮件，请立即报告管理员
- 定期更换密码以增强安全性

### ❌ 避免的做法

- 不要将验证码提供给声称是客服人员的人
- 不要在非Omniflow官方网站的任何地方输入验证码
- 不要将OTP邮件转发给他人
- 不要截图OTP验证码

## 故障排除

| 问题 | 解决方案 |
|---------|--------|
| 未收到OTP邮件 | 检查垃圾邮件/广告邮件文件夹，确认邮箱地址正确，联系管理员 |
| OTP邮件进入促销/垃圾邮件 | 检查邮箱设置，将发件人标记为可信联系人 |
| 来不及输入验证码时间就到了 | 点击"重新发送OTP"获取新验证码 |
| 多次输入错误验证码 | 等待5分钟，确认输入正确后再提交 |
| 停留在当前页面 | 刷新浏览器，如果问题仍然存在，请退出登录后重新登录 |

## 开发说明（开发模式）

### 开发环境跳过功能

在开发环境中，管理员可以启用2FA跳过功能：
- 启用后，不需要输入OTP验证码
- 用户验证密码后可直接登录
- 通常用于测试或演示

**重要提示**：此功能不得在生产环境中启用，因为会降低安全性。
